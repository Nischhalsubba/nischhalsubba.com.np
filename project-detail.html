Web3 / Fintech Website Redesign Protocol & Wallet 2024

Mokshya Protocol Mokshya Protocol

Website redesign and product UX for an open-source Web3 protocol. I translated a complex governance story into clear public messaging, then aligned wallet onboarding and transaction flows with the same trust-first language.

View Mokshya
Role

Lead Product Designer
Research, Narrative, UI, Prototyping

Team

2 Engineers, 1 PM, 1 Researcher

Timeline

Aug 2024 - Dec 2024 (5 Months)

Outcome

Clear protocol narrative and safer onboarding

THE PROBLEM

The protocol story was hard to grasp, and the product UI felt opaque.

The public website did not explain what Mokshya was or why it mattered. Inside the product, early users struggled to understand what was happening behind each wallet action. Governance and staking flows were powerful but felt opaque, and the interface did not clearly explain risk, fees, or outcomes.

"If users cannot see what happens next, they will not sign." - Stakeholder feedback

Goals & Success Metrics

Goal
Onboarding Clarity

Explain key terms, fees, and outcomes before signing any transaction.

Goal
Governance Confidence

Make proposals, voting, and staking feel transparent and predictable.

Goal
Scalable UI System

Create reusable components for future protocol modules.

Constraints & Reality Check

  • Security First: Transaction signing and risk messaging had to be explicit and audit-friendly.
  • Technical Reality: Wallet adapters and chain confirmations introduced edge cases that the UI needed to explain.
  • Education Gap: The product had to reduce jargon without oversimplifying protocol mechanics.
WEBSITE REDESIGN

Translate open-source innovation into a clear story.

The public site needed to explain the protocol in one scroll: what it is, why it matters, and how people can participate. I restructured the narrative, simplified the jargon, and aligned the visual system with the product UI so trust carried through every touchpoint.

  • Rewrote hero messaging and value propositions to reduce cognitive load.
  • Defined a modular section system for roadmap, governance, and community.
  • Built a token-driven UI kit so marketing and product screens shared the same language.
INFORMATION ARCHITECTURE

A safer path from intent to signature

The new flow separates education from action. Users first understand what a transaction does, then review fees and outcomes, and only then sign. Governance and staking share a consistent layout to reduce cognitive load.

[Flow Diagram: Explore → Learn → Review → Sign → Track]
DESIGN SYSTEM

Built for Protocol Scale

Component Library

A token-first UI system ensured every surface, from wallet cards to governance tables, used the same language and spacing rules.

  • Transaction Review Panels with clear risk states
  • Governance Tables with proposal clarity and filters
  • Wallet Cards with balance, status, and action hierarchy

Final Implementation

Delivery included a tokenized design system, interactive prototypes for high-risk flows, and implementation-ready specs aligned with engineering constraints.
